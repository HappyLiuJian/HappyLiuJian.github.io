  1. 定宽元素水平居中 {width: xxpx; margin: xxpx auto;}
  2. 不定宽元素居中 {position:absolute top:xxpx right:xxpx ….} 其父辈是position: relative的
  3. 文字垂直居中 {height:xpx; line-height:xpx} 即高度与行间高度一致，这个方法对行内元素和会级元素都有用
    {padding:xxpx;} 外框元素设置padding值
或者采用position方法，进行精确定位
注意overflow方法的使用不然可能会导致内部元素溢出
  4. 默认样式一定要预先清除···一定要 *{margin:0; padding:0;}
  5. overflow:hidden特别有用···特别有用
  6. 先考虑好元素的语义结构很重要
  7. text-indent（文本缩进用，text-indent替换padding-left，是为了防止菜单长度增加。例如：padding-left:10px; 会导致宽度也增加10px ）
  8. 选择器a b代表a中的b，而a,b才代表a和b都具备相应样式
  9. float会挨着前一个float对象元素进行float，而absolute在不设置偏移的情况下会抢占前一个absolute的元素的左上角的位置，把前一个元素给遮盖
  10. absolute属性在没有设置偏移量的时候，出现在最初的位置，这个最初的位置是指父级元素的内容左上角（会根据父元素的padding等属性可能离父元素有距离），这里可以视作偏移的默认位置，之后设置偏移量，按照祖先元素中有定位的作为参考点，哪个方向设置多少就距离那个参考点多少，没有设置的方向，则就是默认的值，fixed与其基本相同，只是参考点一定为html文档
  11. 行内元素没有width、height两个属性，margin和padding也只有左右方向有效果，上下都无效，设置其盒子高度可以通过line-height来设置

  12. 垂直方向margin属性的重叠
    已知 div 高 35px，紧邻其后的 X 元素上外边距为 15px，为该 div 设置 fixed 定位后，div与浏览器可视窗口上边界相距15px。
    为什么呢？
    因为X元素15px的上外边距与body的上外边距信息重叠了，所以body也跟着下移；然后，div在父元素body内，且没有设置偏移量，所以div相对其父元素body进行绝对定位（？）。

  13. 要使得ul水平，不仅可以利用float:left 或者absolute方法，也可以使用diplay:inline-block的方法来使得ul列表水平排列

  14. textarea固定大小/禁止拖动的方式
    resize: none; //彻底关掉拖动功能
    width:xxpx height:xxpx max-width:xxpx max-height:xxpx //不能拖动但是拖动图标还在

  15. 对于position为absolute或者fixed的标签元素，常常使用left: 50%; margin-left: xxpx;的技巧对他进行定位操作
    //html5实现锚点只能使用href=”#id”,不再使用name了

  16. 背景图片必须设置了宽度和高度才会显示出来，并且背景图片不像img标签那样可以伸缩。在不考虑浏览器兼容性的条件下，实现背景图片自适应窗口大小可以采用background-size属性，如:

    background-image: url(a.jpg);
    width: 1000px;

  17. 对于chrome来说，它的input标签在点击的时候都会有一个外部border和发光效果，这时候可以通过设置outline属性为空来取消这个效果

  18. 在css中，widht和height指的是内容区域的宽度和高度。增加内边距，边框和外边距不会影响内容区域的尺寸，但是会增加元素框的总尺寸。

  19. 浮动会让元素塌陷。即被浮动元素的父元素不具有高度。例如一个父元素包含了浮动元素，它将塌陷具有零高度。你可以按以下方法处理：

    在浮动元素后加个div设置clear: both; height:0,overflow:hidden

    设置父元素浮动;

  20. ie6不支持min-height一类的属性，这时候要做css hacking，即使用_hright属性。使用min-height属性是为了在浮动或者决定定位的会级元素没有内容时也能有一定的高度而不至于塌下去

  21. 当既有背景图片，又有背景颜色的时候，并且背景图片不重复，就在左边显示背景图片，没有背景图片的地方显示背景颜色。这可以通过盒子模型的3d图来理解

  1. 使用js改变某个元素的样式一般不采用x.style.x的方式，而是通过改变元素的类名，如设置active,on等类名，向这些类中添加属性，从而实现改变元素样式

  2. 鼠标的位置保存在event中，clientX和clientY表示其x、y坐标

  3. 有三种方法能够确定浏览器窗口的尺寸（浏览器的视口，不包括工具栏和滚动条）。

对于Internet Explorer、Chrome、Firefox、Opera 以及 Safari：

window.innerHeight - 浏览器窗口的内部高度
window.innerWidth - 浏览器窗口的内部宽度
对于 Internet Explorer 8、7、6、5：

document.documentElement.clientHeight
document.documentElement.clientWidth
或者

document.body.clientHeight
document.body.clientWidth

  1. var that = this;这个语句存在的意义是：
    a.add(a,b);这种方式调用函数相当于在a对象里添加一个方法为add，这个add方法可以内部可以引用this访问a对象，也就是说，这个this绑定到了a;相反，var a = add(a,b);这种方式访问函数add就不是这样了，这里的add依然只是一个函数，不是对象的方法，它的this绑定到window对象(或者它的父对象),此时会出现一种情况

    var out = {

    a : function add() {
    //...
    var b = function b() {
        this;        //指向window
    }
}

    }

也即是说：b不能通过this访问out对象，这是一个js设计缺陷，此时可以使用var that = this;来解决

var out = {
    a : function add() {
        //...
        var that = this;
        var b = function b() {
            this;        //指向window
            that;        //指向out对象
        }
    }
}

如上，在外部方法中给一个新变量that赋值this，此时内部函数b可以通过其外部函数a的作用域链访问that变量，也就是外部out对象

  1. document指整个文档 document.documentElement指元素html document.body指元素body 可以理解为在没有querySelector之前的选择器工具替代

实现对象继承

javascript中的函数并没有方法签名(函数名)，所以不能实现接口继承只能支持实现继承

原型链

将子原型对象等于父类型的实例，此时这个原型对象包含指向父类型对象的指针

function SuperType() {                        /*使用构造函数和原型组合的方法构造对象*/
    this.name = "liujian";
}

SuperType.prototype.getName = function() {
    return this.name;
}

function subType() {
    this.age = 20;
}

subType.prototype = new SuperType();        //子原型对象等于父类型的实例,实现原型链继承关系

subType.prototype.getAge = function() {        //这里不能用对象字面量的形式书写subType.prototype对象，否则整个对象都会重写，导致原型链破坏
    return this.age;
}

/*不能写为
subType.prototype = {
    getAge : function() {
        return this.name;
    }
}*/

var a = new subType();
alert(a.getName());                //"liujian"

和原型模式的缺点一样，继承会导致每一个子对象实例都共享父对象类型中的引用类型的值(比如Array等)

借用构造函数

function SuperType() {
    this.colors = ["red","green"];
}

function Subtype() {
    SuperType.call(this);        //利用call或者apply改变函数的作用域，实现超类型的构造函数创建
}

var i1 = new Subtype();
i1.colors.push("yellow");
alert(i1.colors);            //"red","green","yellow"
var i2 = new Subtype();
alert(i2.colors);            //"red","green"

使用构造函数的优点是可以传递参数，但是函数复用就无效了

组合继承

包含了二者的优点，与组合构造一样使用最多

function SuperType(name) {
    this.name = name;
    this.colors = ["red","green"];
}

SuperType.prototype.sayName = function() {
    return this.name;
};

function Subtype(name,age) {
    SuperType.call(this,name);        //利用call或者apply改变函数的作用域，实现超类型的构造函数创建
    this.age;
}

Subtype.prototype = new SuperType();    //继承方法

Subtype.prototype.sayAge() {
    return this.age;
}

var i1 = new Subtype("liujian",20);
i1.colors.push("yellow");
alert(i1.colors);            //"red","green","yellow"
i1.sayName;                    //"liujian"
i1.sayAge;                    //20
var i2 = new Subtype("gene",20);
alert(i2.colors);            //"red","green"
i2.sayName;                    //gene
i2.sayAge;                    //20

原型式继承

借助已有的对象创建新对象

function object(o) {            //函数的作用是传入一个已有的对象，返回一个把这个已有对象作为原型的新对象
    function F() {}
    F.prototype = o;
    return F{};
}

var person = {
    name : "gene";
    friends : ["liujian","hah"];
};

var Person = object(person);    //相当于创建了person对象的两个副本对象
Person.name = "gene";
Person.friends.push("jji");

var Person2 = object(person);
Person2.name = "bob";
Person2.friends.push("lini");

alert(person.friends);        //"liujian","hah","jji","lini"

可以利用object.create()方法规范原型式继承

寄生式继承

把继承过程包裹在一个函数内部

function createAnother(original) {
    var clone = object(original);
    clone.sayHi = function() {
        alert("hi");
    };
    return clone;
}

寄生组合式继承

组合继承的不足在于无论什么情况下都会调用两次父类型的构造函数（一次在Subtype.prototype = new SuperType();，另一次在SuperType.call(this,name);

function inheritPrototype(subType,superType) {
    var prototype = object(superType.prototype);        //创建父类型的副本
    prototype.constructor = subType;                    //为副本添加constructor，弥补重写原型失去的默认constructor值
    subType.prototype = prototype;                        //指定对象副本赋值给子类型的原型，实现继承
}

function SuperType(name) {
    this.name = name;
    this.colors = ["red","green"];
}

SuperType.prototype.sayName = function() {
    return this.name;
};

function Subtype(name,age) {
    SuperType.call(this,name);        //利用call或者apply改变函数的作用域，实现超类型的构造函数创建
    this.age;
}

inheritPrototype(SubType,SuperType);    //新的寄生组合形式继承方法
//Subtype.prototype = new SuperType();    //原来的继承方法

Subtype.prototype.sayAge() {
    return this.age;
}

var i1 = new Subtype("liujian",20);
i1.colors.push("yellow");
alert(i1.colors);            //"red","green","yellow"
i1.sayName;                    //"liujian"
i1.sayAge;                    //20
var i2 = new Subtype("gene",20);
alert(i2.colors);            //"red","green"
i2.sayName;                    //gene
i2.sayAge;                    //20

在sublime中的安装

首先你需要为sublime text安装Package Control组件:

  • 按Ctrl+`调出sublime text的console

  • 粘贴以下代码到底部命令行并回车

    import urllib2,os;pf=’Package Control.sublime-package’;ipp=sublime.installedpackagespath();os.makedirs(ipp) if not os.path.exists(ipp) else None;open(os.path.join(ipp,pf),’wb’).write(urllib2.urlopen(‘http://sublime.wbond.net/'+pf.replace(‘ ‘,’%20’)).read())

  • 重启Sublime Text

  • 在Perferences->package settings中看到package control，则表示安装成功

使用Package Control安装Emmet插件:

  • 按Ctrl+Shift+P命令板
  • 输入pci然后选择install Package，然后输入emmet找到 Emmet Css Snippets，点击就可以自动完成安装
